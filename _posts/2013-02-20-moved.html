My choice of [jekyll on] github pages was because I wanted something I can control - something that resides right here on my pc; something I can easily break, move and mess around with. Fuck that. I chose github pages because it's like some cool 'hacky' thing. And 'hacky' is fun. But there are things I will miss. Autosharing on social networks and posting via email as examples.

Setting up Jekyll on Github pages

  • Create your page

    To create your page, simply create a new github repo with the name: [your username].github.com

  • Clone the repo

    Cd to your choice directory and clone the repo with any name you want. I used blog

    {% highlight %} $ git clone https://github.com/[your username]/[your username].github.com.git blog {% endhighlight %}

    I'm cloning over https because my current connection sucks on ssh.

  • Install jekyll

    Install jekyll. I assume you have ruby installed already. If not, that comes first.

    {% highlight %} $ sudo gem install jekyll {% endhighlight %}

    This installs jekyll and other dependencies. Really, if you know what you are doing (which you won't as a first timer), you may not need to install jekyll locally as it is github that eventually run their server version on your pages. It is however necessary to run a local version for test, debug and development purpose.

  • Setup your blog

    The jekyll blog has a standard structure of folders and files. Mine has just _layouts (the standard folder that contains layouts), _posts (the standard folder that contains posts), assets (contains css and images), index.html, posts.html and the config file _config.yml. See usage for details about creating this and content.

    My _config.yml file has just two lines. Pygments is for code highlighting.

    {% highlight yaml %} pygments: true pagination: 20 {% endhighlight %}

    I migrated my posterous posts using the posterous migrator at that came with Jekyll.

    {% highlight %} $ ~ {% endhighlight %}
